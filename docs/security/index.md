## 安全运维要点

### 访问控制

- **最小权限原则**：仅授予运维人员完成工作所需的最小权限。
- **多因素认证**：对关键系统启用 MFA（多因素认证）。
- **堡垒机接入**：所有运维操作必须通过堡垒机进行，禁止直接 SSH 登录。
- **会话超时**：闲置会话自动断开，防止未授权访问。

### 审计与追溯

- **操作审计**：所有运维操作需记录操作人、操作时间、操作内容。
- **日志保留**：操作日志保留不少于 180 天，满足合规要求。
- **定期审计**：每月审查运维操作记录，发现异常行为。

### 合规要求

- 满足等保 2.0 三级安全要求。
- 满足 GDPR 或《数据安全法》相关数据保护规定。
- 行业合规（如金融行业的 PCI DSS、银保监会要求）。

### 日常运维安全规范

| 规范项 | 说明 |
|-------|------|
| 密码管理 | 使用密码管理器，定期轮换，禁止明文传输 |
| 基线检查 | 每月对服务器进行安全基线检查 |
| 漏洞修复 | 高危漏洞 24 小时内修复，中危 7 天内修复 |
| 变更管理 | 所有变更需审批，留存变更记录 |
| 备份策略 | 关键数据每日备份，全量备份每周一次 |
