# rsync + inotify 实时文件同步工具

**一条命令完成所有配置**，支持本地服务器和备份服务器两种角色。

---

## 核心概念（重要！）

```
┌─────────────────────────────────┐          ┌─────────────────────────────────┐
│       本地服务器（数据源）        │          │       备份服务器（接收端）        │
│                                 │          │                                 │
│  • 监听文件变化                   │          │  • 接收推送的文件                 │
│  • 文件变化后自动推送             │  ──────▶  │  • 提供 rsync 服务               │
│  • 运行【模式1】                  │          │  • 运行【模式2】                  │
│                                 │          │                                 │
│  示例: 192.168.1.10              │          │  示例: 192.168.1.20              │
└─────────────────────────────────┘          └─────────────────────────────────┘

                    数据流向：本地服务器 ──推送──▶ 备份服务器
```

---

## 快速开始

### 第一步：在备份服务器上运行

```sh
# 下载脚本
curl -O https://your-server/rsync-inotify-sync.sh
chmod +x rsync-inotify-sync.sh

# 执行脚本
./rsync-inotify-sync.sh
```

选择 **【模式 2：备份服务器】**

### 第二步：在本地服务器上运行

```sh
# 下载脚本
curl -O https://your-server/rsync-inotify-sync.sh
chmod +x rsync-inotify-sync.sh

# 执行脚本
./rsync-inotify-sync.sh
```

选择 **【模式 1：本地服务器】**，输入备份服务器显示的连接信息。

## 常见问题

### 1. 连接失败

检查以下几点：

- 备份服务器 rsync 服务是否启动：`systemctl status rsyncd`
- 防火墙是否开放 873 端口
- 模块名是否正确（不是路径）
- 用户名密码是否一致

### 2. 模块名称错误

**错误**：输入了路径如 `/www/backup/`  
**正确**：输入模块名如 `webdata`

模块名是在备份服务器的 `/etc/rsyncd.conf` 中定义的 `[模块名]`

### 3. 日志显示 `name lookup failed`

这个警告不影响同步功能。如果想消除，在备份服务器添加 hosts 映射：

```sh
echo "1.10.10.54  local-server" >> /etc/hosts
```

或在 `/etc/rsyncd.conf` 全局配置中添加：

```
reverse lookup = no
```

然后重启服务：`systemctl restart rsyncd`

### 4. 验证同步是否正常

在本地服务器创建测试文件：

```sh
echo "test" > /var/www/html/test.txt
```

在备份服务器检查：

```sh
ls -la /data/backup/
# 应该能看到 test.txt
```

## 文件位置

|文件|本地服务器|备份服务器|
|--|--|--|
|配置文件|`/opt/rsync-sync/sync.conf`|`/etc/rsyncd.conf`|
|密码文件|`/etc/rsync.password`|`/etc/rsyncd.secrets`|
|日志文件|`/opt/rsync-sync/sync.log`|`/var/log/rsyncd.log`|

## 注意事项

1. **需要 root 权限**运行
2. **先配置备份服务器，再配置本地服务器**
3. **模块名不是文件路径**
4. 两边的用户名和密码必须一致
5. 这是**单向同步**：本地服务器 → 备份服务器
