# 静态路由冗余配置故障排查与解决方案

## 问题现象

- **配置**：为同一目标网络配置了多条下一跳不同的静态路由，意图实现负载分担（ECMP）或主备冗余。
- **故障**：其中一条链路的中间环节发生中断，但连接此链路的本地交换机物理端口仍保持 UP 状态。
- **结果**：流量持续向故障链路转发，造成业务中断，路由未能实现自动切换。

## 根本原因

- 静态路由本身无智能检测机制，其生效仅依赖于本地出接口的物理状态。
- 对于"端口 UP 但下一跳不可达"的中间链路故障，静态路由无法感知，导致流量黑洞。

## 解决方案：部署路径检测协议

### 方案一：BFD（双向转发检测）— 强烈推荐

**机制**：轻量化的毫秒级故障检测协议，通过快速收发检测报文监控路径连通性。

**优点**：收敛速度极快（通常 < 1秒），资源开销小。

**核心配置命令（华为交换机）**：
```bash
# 1. 创建BFD会话
bfd
bfd to_router1 bind peer-ip 192.168.1.1 source-ip 192.168.1.2
discriminator local 10
discriminator remote 20
commit
quit

# 2. 将静态路由与BFD会话绑定
ip route-static 1.10.5.0 255.255.255.0 192.168.1.1 track bfd-session to_router1
```

### 方案二：NQA（网络质量分析）— 备选方案

**机制**：通过模拟 ICMP 等测试报文主动探测目标 IP 的可达性。

**优点**：可单端部署，无需对端设备支持。

**缺点**：检测速度（秒级）慢于 BFD。

**核心配置命令（华为交换机）**：
```bash
# 1. 创建NQA测试实例
nqa test-instance admin nqa_test1
test-type icmp
destination-address ipv4 192.168.1.1
frequency 10
interval seconds 5
probe-count 2
start now
quit

# 2. 将静态路由与NQA实例绑定
ip route-static 1.10.5.0 255.255.255.0 192.168.1.1 track nqa admin nqa_test1
```

## 总结与行动建议

1. **立即行动**：联系对端网络负责人，协商部署 BFD。
2. **设计原则**：所有用于冗余的静态路由必须与 BFD 联动。
3. **应急措施**：若对端暂不支持 BFD，可临时采用 NQA 方案作为过渡。
